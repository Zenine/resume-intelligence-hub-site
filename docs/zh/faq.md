# 常见问题

### resume-intelligence-hub 是什么？

resume-intelligence-hub 是一个 AI agent skill，帮你搭建一个个人职业发展中枢——一个私有 Git 仓库，既是简历、面试准备、课题申报、投递前核查的单一事实来源，也是你的职业指南针：命名跳一级目标、盘差距、写季度 SMART 计划。

### 它具体解决什么问题？

大多数人的职业管理是被动的：简历散落在各个文件夹，面试前临时抱佛脚，没有系统化地追踪向目标职位迈进的进度。resume-intelligence-hub 把这些换成结构化基础设施：`profiles/` 存放事实性职业数据，`AGENTS.md` 顶部锁定定位目标让每份生成的简历都朝它偏置，六个工作流处理从 JD 定制简历到投递前公开资料核查的一切。

### 适合谁用？

任何正在求职、计划往高 1-2 级跳槽、或申请科研课题的人——不限行业，不限职级。尤其适合：需要兼顾中英文双语市场、同时有招聘投递和科研申报两条线、或者希望 AI 记住你的职业背景而不是每次从头开始的用户。

### 和其他简历 AI 工具有什么不同？

大多数简历工具是一次性的：贴 JD 出简历。resume-intelligence-hub 是可持续迭代的基础设施，跨越数月的求职过程。定位锁让所有输出偏向你的目标；核查工作流在投递前用公开资料核实每一条承重声明；差距分析精确识别你需要补的技能、资历、管理范围和人脉。一次性专项工具（ATS 优化、LinkedIn 主页生成器）可以叠在这个 hub 上处理特定场景。

### 支持我用的 AI IDE 吗？

支持。hub 用 `AGENTS.md` 作为指令文件——这是跨 IDE 通用标准。Claude Code、Cursor、Codex、Cline、Windsurf 和 GitHub Copilot 都能读它（某些 IDE 需要软链或复制一次）。具体配置方式见[快速开始 — 常见问题排查](./quick-start.md#常见问题)。

### 科研课题那条线是必须的吗？

不是，默认关闭。初始化时 AI 会问你是否申请学术课题或科研经费。回答否，课题相关文件（`profiles/research.md`、`科研/`、课题申报工作流）就完全不生成。大多数用户不需要。

### 怎么开始？

一条命令安装，一句话触发：

```bash
npx skills add Zenine/resume-intelligence-hub -g -y
```

然后打开 AI IDE，进入你想放 hub 的目录，说：

> **帮我搭个简历库**

AI 问你 7 个问题，10 分钟内搭好你的私有 hub。完整流程见[快速开始](./quick-start.md)。
